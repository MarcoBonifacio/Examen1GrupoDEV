Clínica "Salud Conectada"

Implementación de Arquitectura Cliente-Servidor

Desarrollado por el Grupo D.E.V. (Diseño de Software - 2026)

🏥 El Problema y la Solución

Problemática Actual

La administración manual en el sector salud genera redundancia de datos, pérdida de información, conflictos de horarios y tiempos de espera prolongados.

Solución Tecnológica

Digitalizar y automatizar la gestión clínica mediante un sistema que centraliza la información para registrar pacientes, médicos y coordinar citas en tiempo real.

⚙️ Arquitectura del Sistema

Arquitectura 2-Tier (Cliente-Servidor)

El sistema se construyó en dos niveles. La capa de presentación (interfaz gráfica) y la lógica de acceso a datos coexisten en el código de la aplicación de escritorio (code-behind).

El servidor de base de datos se encarga de procesar y almacenar transacciones de forma exclusiva.

🗄️ Base de Datos y Entidades (CRUD)

La base de datos relacional BDClinica está normalizada y se gestiona en SQL Server, relacionando un paciente a muchas citas (1:N) y un médico a múltiples citas (1:N).

👥 Pacientes

Gestión de altas, información de contacto, demografía, historial y búsquedas por identificador.

⚕️ Médicos

Registro de profesionales de la salud, actualización de sus datos y gestión de especialidades.

📅 Citas

Programación de encuentros médicos, asociando pacientes y médicos. Permite reprogramar y cancelar.

💻 Tecnología y Flujo de Operaciones

C# y Windows Forms ADO.NET SQL Server Procedimientos Almacenados Control DataGridView

Ciclo de vida de una operación:

1
El usuario interactúa con la interfaz gráfica en Windows Forms (ej. "Guardar Cita").
2
El evento del botón captura los datos ingresados en la pantalla.
3
ADO.NET inicializa SqlConnection abriendo la red con el servidor SQL.
4
SqlCommand ejecuta Procedimientos Almacenados precompilados evitando inyección SQL y mejorando el rendimiento.
5
La interfaz recibe los resultados y actualiza dinámicamente el control DataGridView.
